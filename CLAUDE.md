# CLAUDE.md — はちバス時刻表（八戸市営バス 時刻表検索PWA）

## プロジェクト概要

八戸市営バスのGTFS-JPオープンデータを使った、個人用の時刻表検索Webアプリ。
乗車バス停と降車バス停を指定して時刻を調べる検索機能と、定番ルートの
次便を起動即表示するダッシュボードを持つ。GitHub Pagesで公開し、
スマホのホーム画面に追加してPWAとして使う。

- 利用者は開発者本人のみ（汎用公開アプリではない）
- 最優先UX:「起動して0タップで、定番ルートの次のバスまでの残り時間が見える」
- サーバー不要。静的ファイルのみで完結させる

## データソース

- 八戸市交通部が公開するGTFS-JP（標準的なバス情報フォーマット）
- zipファイルを `gtfs/` に手動で配置してあるので、解凍して読むこと
- 主要ファイル: stops.txt, stop_times.txt, trips.txt, routes.txt,
  calendar.txt, calendar_dates.txt
- 文字コードはUTF-8（BOM付きの可能性あり。`utf-8-sig` で読む）

## 前処理: scripts/build_data.py

Python 3標準ライブラリのみで実装（pandas等の外部依存は不可。
更新作業をどのPCでも実行できるようにするため）。

要件:

1. GTFSを読み、便（trip）ごとに [stop_id, 発時刻] の列を圧縮した
   JSONを生成する。stop_times.txt は数万行になるため、出力は
   数MB以内を目標に以下の工夫をする:
   - stop_id・trip_id は連番の整数に振り直す
   - 時刻は "HH:MM:SS" 文字列ではなく深夜0時からの分数（整数）にする
   - キー名は1〜2文字に短縮する
2. calendar.txt / calendar_dates.txt から各便のサービスパターンを
   「平日 / 土曜 / 日祝」の3区分に分類して付与する。
   分類できない特殊ダイヤがあれば、最も近い区分に寄せた上で
   ビルドログに警告を出す
3. calendar_dates.txt の内容（祝日・特別ダイヤの例外日）は
   日付→サービス区分の辞書として data.json に含め、アプリ側で
   「今日がどのダイヤか」の判定に使えるようにする
4. 同名バス停ののりば違い（例: 「八日町」の1番のりば・2番のりば）を
   統合検索できるよう、バス停名ベースのグループ情報も出力する
5. 「25:10」のような24時超え表記に対応する（分数化で自然に扱える）
6. stops.txt の stop_name に加えて、可能なら読み仮名
   （stop_name_kana 等の拡張列があれば）も出力し、かな検索に使う
7. calendar.txt の end_date の最大値を「ダイヤ有効期限」として
   data.json に含める
8. 出力先: docs/data.json

## アプリ本体: docs/index.html

単一HTMLファイル。vanilla JS。CDN・外部ライブラリ・Webフォント不使用。
スマホ優先レイアウト（幅375px基準、max-width 420px 中央寄せ）。

### デザイン

- **docs/mock.html のデザインを忠実に再現すること。**
  配色・角丸・タブ様式（下線タブ）・結果リスト様式（罫線リスト）・
  次便カード（青グラデーション）はモック準拠
- カラーパレット: 市バス青 #1656A0 / 濃紺 #0E3A6E / アンバー #FFB100 /
  背景 #EEF1F4 / 罫線 #DDE2E8
- 数字は font-variant-numeric: tabular-nums で桁揃え

### 画面構成（上から）

1. **ヘッダー**: アプリ名、今日の日付と適用ダイヤ
   （例:「7/11（土）土曜ダイヤ」。calendar_dates の例外日を優先判定）
2. **定番ルートチップ**: 登録済みの乗降ペアを横スクロールで並べる。
   タップで下の次便カードが切り替わる。「＋追加」チップあり
3. **次便カード**（青グラデーション）: 選択中の定番ルートの次便を
   「あと◯分」の大きな数字で表示。系統番号バッジ・発着時刻・
   所要時間・行き先も表示。1分ごとに自動更新
4. **後続便**: 次便の後の2便を小さいカードで横並び表示
5. **検索セクション**（常時表示、折りたたまない）:
   - 乗車・降車バス停のテキスト入力。部分一致で候補をリスト表示
     （漢字・ひらがな・カタカナいずれでもヒット。カナ⇄かなは正規化）
   - 乗車⇄降車の入れ替えボタン
   - 時刻指定: 「今から / 出発 / 到着」の下線タブ ＋
     時刻入力欄（type="time"）
     - 今から: 現在時刻以降の便（時刻欄は無効化）
     - 出発: 指定時刻以降に出発する便を早い順
     - 到着: 指定時刻までに到着する便を、到着が指定時刻に近い順
       （＝一番遅く出られる便が上）
   - ダイヤ切替: 「平日 / 土曜 / 日祝」の下線タブ。初期値は今日
   - 結果一覧の上に検索条件キャプション
     （例:「16:30 までに到着する便」）
6. **検索結果**: 罫線リスト。各行に系統番号バッジ・乗車時刻→降車時刻・
   所要分・行き先・★ボタン。両停留所をその順で通る便のみ抽出。
   「今から」モードでは直近便を左青バー＋薄青背景でハイライト
7. **フッター**: 出典表記「八戸市交通部 GTFS-JPオープンデータ」と
   ダイヤ有効期限

### 定番ルート機能

- 検索結果の★タップで定番ルートに登録。localStorageで管理
  （コード直書きにしない）。チップ長押しで削除（確認ダイアログあり）
- 定番が複数あるとき、午前は「行き」・午後は「帰り」に相当する
  ペアを自動選択する（同一2停留所の往復ペアが登録されている場合、
  午前=先に登録した方向、午後=その逆、を初期表示。単純でよい）
- 最後に見ていた定番ルート・検索条件も localStorage に保存し、
  次回起動時に復元する

### 検索ロジック

- 乗車停グループと降車停グループの両方を「その順序で」通る trip を
  抽出する（stop_sequence 比較）
- のりば違いは統合して検索し、結果には実際ののりば名を表示する
- 24時超えの便は当日ダイヤの深夜便として扱う
- 選択中のダイヤ区分（平日/土曜/日祝）でフィルタする

### PWA対応

- manifest.json: name「はちバス時刻表」、standalone、theme_color は
  市バス青 #1656A0。アイコンは 192px / 512px をPillowか手書きSVG→PNG
  変換で生成する（バスを模した簡単な図案でよい）
- sw.js: キャッシュファースト戦略。index.html / data.json /
  manifest.json / アイコンをプリキャッシュ
- **キャッシュ名にバージョン番号を含める**（例: `hachibus-v3`）。
  data.json を更新したら必ずこの番号を上げ、activate イベントで
  旧キャッシュを削除する仕組みにする
- 起動時に data.json のダイヤ有効期限をチェックし、期限まで14日を
  切ったら画面上部に「ダイヤ改正の可能性があります。データを
  更新してください」と警告バーを表示する

## README.md

ダイヤ改正時の更新手順を必ず記載する:

1. GTFS-JPオープンデータ一覧サイト（bustime.jp のGTFS一覧）から
   八戸市営バスの最新zipをダウンロードし、`gtfs/` に上書き配置
2. `python scripts/build_data.py` を実行
3. `sw.js` のキャッシュバージョン番号を1つ上げる（更新箇所の行を
   READMEに明記しておく）
4. git add → commit → push（GitHub Pagesに自動反映）
5. スマホでアプリを開き直し、フッターの有効期限が更新されたことを確認

初回セットアップ手順（GitHub Pages の Settings → Pages →
main ブランチ /docs を選ぶ）も記載する。

## 検証（実装中に必ず行うこと）

1. build_data.py 実行後、出力された data.json のサイズと
   バス停数・便数をログに出す
2. 実在の2停留所ペアで抽出テストを行い、便数・最初と最後の便の
   時刻をコンソールで確認してから index.html の実装に進む
3. 平日/土曜/日祝それぞれで結果が変わることを確認する
4. 「到着」モードで並び順（指定時刻に近い着が先頭）を確認する
5. Node.js があれば `node --experimental-default-type=module` 等での
   検索ロジック単体テスト、なければブラウザのconsoleで確認する
